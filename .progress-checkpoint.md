## Architecture

```text
Request ‚Üí ObservabilityMiddleware (FastAPI)
           ‚îú‚îÄ‚îÄ Generate correlation_id ‚Üí contextvars
           ‚îú‚îÄ‚îÄ time.perf_counter() start/end
           ‚îî‚îÄ‚îÄ Persist RequestLog

        ‚Üí LangGraph StateGraph (thread_id ‚Üí SqliteSaver persistence)
           ‚îú‚îÄ‚îÄ Agent Node ‚Äî calls litellm.completion() directly
           ‚îú‚îÄ‚îÄ Tool Node ‚Äî executes 3 tool functions
           ‚îî‚îÄ‚îÄ ObservabilityCallback (reads correlation_id from contextvars)
                ‚Üí AiObservabilityLog (linked to RequestLog via correlation_id)
```

**Stack:** `litellm` + `langgraph` + `langgraph-checkpoint-sqlite`
**Observability:** Two-tier via `contextvars` ‚Äî FastAPI middleware for request metrics, LangGraph callback for LLM metrics, linked by `correlation_id`
**Removed:** AIProvider ABC, CacheService, AgentSessionManager, MSAgentFrameworkProvider, gRPC (grpcio/grpcio-tools/protobuf)

## Decisions made

- **AI stack:** Chose `litellm` over `agent-framework-openai` ‚Äî Python community standard for multi-provider LLM abstraction. LangGraph handles agent orchestration (state management, tool loop, session persistence).
- **No AIProvider ABC:** LangGraph nodes call `litellm.completion()` directly. Single-provider commitment (litellm) makes the ABC an unnecessary indirection for a demo/POC.
- **No CacheService:** LangGraph's SqliteSaver handles session persistence. AI result caching is YAGNI for demo.
- **No AgentSessionManager:** Replaced by LangGraph thread_id + checkpointer.
- **Observability:** LangGraph `BaseCallbackHandler` instead of background queue worker.
- **Correlation IDs:** `contextvars` (Python stdlib) propagates correlation_id from middleware ‚Üí LangGraph callbacks. Zero extra dependencies.
